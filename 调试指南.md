# 分管理列表调试指南

## 问题描述
添加分管理后，在分管理列表页面无法显示新添加的分管理数据。

## 调试步骤

### 1. 微信开发者工具调试

#### 1.1 查看控制台日志
在微信开发者工具中打开控制台（Console），查看以下日志输出：

```
=== 开始加载分管理列表 ===
请求URL: http://localhost:8081/api/admin/branch-managers
Token: xxx
=== 列表请求成功 ===
响应状态码: 200
响应数据: {...}
获取到的列表数据: [...]
列表长度: x
设置后的页面数据: [...]
```

**关键检查点：**
- 请求URL是否正确
- Token是否存在
- 响应状态码是否为200
- 响应数据中的 `code` 是否为200
- `list` 数组是否有数据

#### 1.2 查看网络请求
在微信开发者工具的"网络"面板中：
1. 筛选 `branch-managers` 请求
2. 查看请求头是否包含 `Authorization`
3. 查看响应体内容

### 2. 后端调试

#### 2.1 查看后端日志
```bash
docker-compose logs -f backend
```

查看是否有SQL查询日志输出：
```
==>  Preparing: SELECT COUNT(*) AS total FROM branch_manager WHERE deleted = 0
==>  Preparing: SELECT id,brand_name,phone,password,status,created_at,updated_at,deleted FROM branch_manager WHERE deleted=0 ORDER BY created_at DESC LIMIT ?
```

#### 2.2 直接测试API
使用 curl 或 Postman 测试：

```bash
# 先登录获取token
curl -X POST http://localhost:8081/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"phone":"13800000000","password":"123456"}'

# 使用返回的token查询列表
curl -X GET "http://localhost:8081/api/admin/branch-managers?page=1&size=10" \
  -H "Authorization: Bearer YOUR_TOKEN_HERE"
```

### 3. 数据库检查

#### 3.1 连接数据库
```bash
# 使用Docker连接MySQL
docker exec -it delivery-mysql mysql -uroot -pmima0221

# 或者使用本地MySQL客户端
mysql -h 127.0.0.1 -P 3307 -uroot -pmima0221
```

#### 3.2 查询数据
```sql
USE delivery;

-- 查看所有分管理
SELECT * FROM branch_manager;

-- 查看未删除的分管理
SELECT id, brand_name, phone, status, created_at 
FROM branch_manager 
WHERE deleted = 0 
ORDER BY created_at DESC;
```

### 4. 常见问题排查

#### 问题1: Token无效或过期
**现象：** 返回401未授权错误
**解决：** 重新登录获取新token

#### 问题2: 数据库连接失败
**现象：** 后端日志显示连接错误
**解决：** 检查MySQL容器是否运行正常
```bash
docker-compose ps
docker-compose logs mysql
```

#### 问题3: 数据被标记为删除
**现象：** 数据库有数据但查询不到
**解决：** 检查 `deleted` 字段是否为0

#### 问题4: 前端数据绑定问题
**现象：** 控制台显示数据但页面不显示
**解决：** 检查 wxml 中的数据绑定语法

### 5. 断点设置位置

#### 后端断点（IDEA）


## 调试流程图

```
开始
  ↓
检查网络请求是否发出 → 否 → 检查前端代码
  ↓ 是
检查响应状态码 → 非200 → 检查后端服务
  ↓ 200
检查响应数据code → 非200 → 检查业务逻辑
  ↓ 200
检查list数组 → 空 → 检查数据库
  ↓ 有数据
检查页面渲染 → 不显示 → 检查wxml绑定
  ↓ 显示
结束
```
