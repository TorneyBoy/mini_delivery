# Mini Delivery - 微信小程序配送管理系统

## 一、项目概述

### 1.1 项目简介
Mini Delivery 是一个基于微信小程序的配送管理系统，主要用于货车司机向水果店和超市配送货品的管理。系统实现了从店铺下单、订单分配、司机拣货到配送完成的完整业务流程。

### 1.2 业务场景
- **应用场景**：水果批发配送业务
- **用户群体**：批发商（分管理）、水果店/超市（店铺）、货车司机
- **核心流程**：店铺下单 → 分配司机 → 司机拣货 → 配送送达 → 账单结算

### 1.3 角色定义

| 角色   | 标识           | 描述                | 核心功能                               |
| ------ | -------------- | ------------------- | -------------------------------------- |
| 总管理 | ADMIN          | 系统超级管理员      | 分管理账号管理、全局数据监控           |
| 分管理 | BRANCH_MANAGER | 品牌方/批发商管理员 | 店铺/司机/商品管理、订单监控、账单生成 |
| 店铺   | SHOP           | 水果店/超市         | 商品浏览、下单、订单管理、账单支付     |
| 司机   | DRIVER         | 货车司机            | 订单分配、拣货、送货、导航             |

### 5.2 订单状态流转

```
┌─────────────────────────────────────────────────────────────────┐
│                        订单状态流转                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   待支付(0) ──支付──▶ 待分配(1) ──分配──▶ 待拣货(2)              │
│       │                                            │            │
│       │                                            ▼            │
│       │                                       待送货(3)          │
│       │                                            │            │
│       │                                            ▼            │
│       └───────────────────────────────▶ 已完成(4) ◀─────────────┘
│                                                                 │
│                              已取消(-1)                          │
└─────────────────────────────────────────────────────────────────┘
```

| 状态值 | 状态名称 | 说明                 |
| ------ | -------- | -------------------- |
| 0      | 待支付   | 订单已创建，等待支付 |
| 1      | 待分配   | 已支付，等待分配司机 |
| 2      | 待拣货   | 已分配司机，等待拣货 |
| 3      | 待送货   | 拣货完成，等待送货   |
| 4      | 已完成   | 送货完成，订单结束   |
| -1     | 已取消   | 订单已取消           |

### 5.3 账单状态

| 状态值 | 状态名称 | 说明                 |
| ------ | -------- | -------------------- |
| 0      | 待支付   | 账单已生成，等待支付 |
| 1      | 已支付   | 账单已支付完成       |

### 5.4 价格类型

| 类型值 | 类型名称 | 说明               |
| ------ | -------- | ------------------ |
| 1      | 老用户价 | 适用于老客户的价格 |
| 2      | 新用户价 | 适用于新客户的价格 |

## 十二、开发规范

### 12.1 前端开发规范

1. **页面开发**
   - 页面位于 `miniprogram/pages/` 目录
   - 每个页面包含 .js、.wxml、.wxss、.json 四个文件

2. **API请求**
   - 使用 `app.request()` 方法进行请求
   - 自动携带Token和处理错误

3. **图片处理**
   - 使用 `app.getImageUrl()` 处理图片URL

4. **样式规范**
   - 使用rpx作为尺寸单位
   - 全局样式在 `app.wxss`

### 12.2 后端开发规范

1. **代码分层**
   - Controller：请求处理
   - Service：业务逻辑
   - Mapper：数据访问

2. **响应格式**
   - 统一使用 `Result` 类返回响应
   - 分页使用 `PageResult`

3. **查询规范**
   - 使用 MyBatis-Plus 的 LambdaQueryWrapper

## 十三、常见问题

### Q1: 数据库连接失败？
检查 Docker 容器是否启动：`docker-compose ps`

### Q2: 小程序请求失败？
1. 检查 `miniprogram/config.js` 中的 `baseUrl` 配置
2. 确保微信开发者工具勾选了「不校验合法域名」

### Q3: 登录返回"用户不存在"？
检查数据库中该用户的 `deleted` 字段是否为 1（已删除）

### Q4: 真机调试网络请求失败？
使用手机热点或真机调试2.0模式

### Q5: 文件上传失败？
检查 `server/data/upload/` 目录是否存在且有写入权限
