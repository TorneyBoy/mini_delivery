# 微信小程序配送管理系统 - 启动指南

## 一、环境要求

- JDK 17+
- Maven 3.6+
- MySQL 8.0+
- Redis (可选，用于缓存)
- 微信开发者工具

## 二、数据库配置

### 1. 启动MySQL服务

**Windows:**
```bash
# 以管理员身份运行CMD
net start mysql
```

**或通过服务管理器启动MySQL服务**

### 2. 创建数据库并初始化数据

```bash
# 方式一：命令行执行
mysql -u root -p < server/src/main/resources/db/init.sql

# 方式二：登录MySQL后执行
mysql -u root -p
source c:/documents/development/mini_delivery/server/src/main/resources/db/init.sql
```

### 3. 验证数据库创建成功

```sql
-- 登录MySQL
mysql -u root -p

-- 查看数据库
SHOW DATABASES;

-- 使用数据库
USE mini_delivery;

-- 查看表
SHOW TABLES;

-- 查看总管理账号
SELECT * FROM admin;
```

### 4. 修改数据库配置（如需要）

编辑 `server/src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/mini_delivery?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root        # 修改为你的MySQL用户名
    password: root        # 修改为你的MySQL密码
```

## 三、启动后端服务

### 1. 进入项目目录

```bash
cd c:/documents/development/mini_delivery/server
```

### 2. 编译项目

```bash
mvn clean compile
```

### 3. 启动服务

```bash
mvn spring-boot:run
```

### 4. 验证启动成功

看到以下日志表示启动成功：
```
Started DeliveryApplication in X.XXX seconds
```

访问 API 文档：http://localhost:8081/doc.html

## 四、启动小程序

### 1. 打开微信开发者工具

### 2. 导入项目

- 点击「导入项目」
- 选择目录：`c:/documents/development/mini_delivery/miniprogram`
- AppID：选择「测试号」或输入你的AppID

### 3. 修改API地址（如需要）

编辑 `miniprogram/app.js`，修改 `baseUrl`：

```javascript
globalData: {
  baseUrl: 'http://localhost:8081/api'  // 确保与后端端口一致
}
```

### 4. 点击「编译」运行

## 六、常见问题

### 1. 数据库连接失败

**错误信息：** `Failed to obtain JDBC Connection`

**解决方案：**
- 确认MySQL服务已启动
- 确认数据库 `mini_delivery` 已创建
- 检查用户名密码是否正确
- 检查端口3306是否被占用

### 2. 端口被占用

**错误信息：** `Port 8081 was already in use`

**解决方案：**
修改 `server/src/main/resources/application.yml` 中的端口：
```yaml
server:
  port: 8082  # 改为其他端口
```

同时修改 `miniprogram/app.js` 中的 `baseUrl`。

### 3. 小程序请求失败

**错误信息：** `request:fail`

**解决方案：**
- 确认后端服务已启动
- 在微信开发者工具中勾选「不校验合法域名」
  - 设置 → 项目设置 → 勾选「不校验合法域名、web-view（业务域名）、TLS版本以及HTTPS证书」

### 4. Redis连接失败（可选）

如果不需要Redis，可以暂时注释掉Redis相关配置。

## 八、API文档

启动后端后访问：http://localhost:8081/doc.html

主要接口：
- POST /api/auth/login - 用户登录
- GET /api/admin/branch-managers - 分管理列表
- GET /api/branch/shops - 店铺列表
- GET /api/branch/drivers - 司机列表
- GET /api/branch/products - 商品列表
- GET /api/shop/products - 店铺商品
- POST /api/shop/orders - 创建订单
- GET /api/driver/pending-orders - 待分配订单
