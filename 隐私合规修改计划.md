# 微信小程序隐私合规修改计划

## 一、当前代码分析

### 1.1 前端登录流程分析

#### 当前入口配置
- [`app.json`](miniprogram/app.json:2) 中第一个页面是 `pages/login/login`
- 用户打开小程序首先看到的是登录页面

#### 当前登录逻辑
- [`app.js`](miniprogram/app.js:111) 中 `checkLogin()` 方法：无token时直接跳转登录页
- [`app.js`](miniprogram/app.js:107) 中 `logout()` 方法：清除token后跳转登录页
- [`app.js`](miniprogram/app.js:138) 中 `request()` 方法：401错误时自动logout

#### 各角色页面登录检查
| 角色           | 页面                                                                                | 登录检查位置 | 当前行为         |
| -------------- | ----------------------------------------------------------------------------------- | ------------ | ---------------- |
| SHOP           | [`pages/shop/index/index.js`](miniprogram/pages/shop/index/index.js:23)             | onLoad       | 未登录跳转登录页 |
| BRANCH_MANAGER | [`pages/branch/index/index.js`](miniprogram/pages/branch/index/index.js:24)         | onLoad       | 未登录跳转登录页 |
| DRIVER         | [`pages/driver/pending/pending.js`](miniprogram/pages/driver/pending/pending.js:14) | onLoad       | 未登录跳转登录页 |

### 1.2 后端认证分析


## 二、违规点分析

### 2.1 严重违规（必须修改）

| 序号 | 违规类型                 | 具体表现                                    | 违规页面/代码                                                        | 规范条款  |
| ---- | ------------------------ | ------------------------------------------- | -------------------------------------------------------------------- | --------- |
| 1    | 一进入小程序就收集手机号 | `app.json` 首页设为登录页，必须登录才能进入 | [`app.json:2`](miniprogram/app.json:2)                               | 禁止情形1 |
| 2    | 浏览功能强制登录         | 店铺用户浏览商品、查看分类需要登录          | [`shop/index/index.js:23`](miniprogram/pages/shop/index/index.js:23) | 禁止情形2 |
| 3    | 普通操作强制登录         | 购物车、个人中心等非必要功能强制登录        | 各页面onLoad中的checkLogin                                           | 禁止情形2 |

### 2.2 合规场景（可保留登录要求）

| 场景           | 原因                 | 合规性 |
| -------------- | -------------------- | ------ |
| 下单/创建订单  | 订单履约需要用户身份 | ✅ 合规 |
| 查看订单列表   | 涉及用户个人数据     | ✅ 合规 |
| 司机接单/配送  | 业务必需的身份验证   | ✅ 合规 |
| 分管理后台管理 | 内部管理功能         | ✅ 合规 |
| 账号注册       | 用户主动触发         | ✅ 合规 |

## 四、详细实施步骤

### 第一阶段：后端接口调整

1. **修改 SecurityConfig.java**
   - 放开商品列表、分类、店铺信息等接口

2. **修改 ShopController.java**
   - 商品列表接口支持无token访问
   - 返回数据中区分登录/未登录状态

3. **新增 GuestController.java（可选）**
   - 提供游客专用的公开接口

### 第二阶段：前端架构调整

1. **创建欢迎页**
   - 新建 `pages/welcome/welcome.*` 文件
   - 设计角色选择界面

2. **修改 app.json**
   - 调整页面顺序，欢迎页为首页
   - 添加新页面配置

3. **修改 app.js**
   - 重构 `checkLogin()` 方法
   - 新增 `requireLogin()` 方法
   - 支持游客模式

### 第三阶段：店铺端改造

1. **修改商品浏览页**
   - 移除强制登录检查
   - 支持游客浏览

2. **修改购物车逻辑**
   - 游客购物车使用本地存储
   - 登录后同步到服务器

3. **添加登录触发点**
   - 结算时检查登录
   - 下单时检查登录
   - 查看订单时检查登录

### 第四阶段：测试验证

1. **功能测试**
   - 游客浏览商品正常
   - 游客加入购物车正常
   - 登录触发时机正确
   - 登录后功能正常

2. **合规检查**
   - 打开小程序不强制登录
   - 浏览功能无需登录
   - 必要场景才触发登录
   - 登录弹窗有明确说明

## 七、注意事项

1. **保持现有功能完整性**：所有现有登录用户的功能不受影响
2. **渐进式改造**：可以先完成核心功能，再逐步优化
3. **用户引导**：添加清晰的操作引导，避免用户困惑
4. **数据安全**：公开接口不暴露敏感信息
5. **审核材料准备**：如需提交审核，准备好使用说明和录屏

## 八、后续优化建议

1. **微信一键登录**：接入微信手机号快速验证组件，简化登录流程
2. **游客数据持久化**：优化游客购物车的本地存储策略
3. **个性化推荐**：基于浏览记录提供商品推荐（无需登录）
4. **登录状态优化**：token刷新机制，避免频繁登录
