## 二、服务器环境搭建（Windows + Ubuntu子系统）

### 2.1 安装WSL2（Windows Subsystem for Linux）

在Windows PowerShell（管理员）中执行：

```powershell
# 启用WSL
wsl --install

# 或者手动安装Ubuntu
wsl --install -d Ubuntu-22.04
```

安装完成后重启电脑，然后设置Ubuntu用户名和密码。

### 2.2 进入Ubuntu环境

```powershell
# 在Windows终端中输入
wsl

# 或者直接打开Ubuntu应用
```

### 2.3 更新Ubuntu系统

```bash
sudo apt update && sudo apt upgrade -y
```

### 2.4 安装Docker

```bash
# 安装Docker依赖
sudo apt install -y ca-certificates curl gnupg lsb-release

# 添加Docker官方GPG密钥
sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

# 添加Docker仓库
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 安装Docker
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# 启动Docker服务
sudo service docker start

# 设置Docker开机自启（可选）
# 在Windows中创建计划任务，开机时执行: wsl -d Ubuntu-22.04 -u root service docker start

# 验证安装
sudo docker --version
sudo docker compose version
```

### 2.5 配置Docker权限（可选，避免每次使用sudo）

```bash
# 将当前用户添加到docker组
sudo usermod -aG docker $USER

# 重新登录Ubuntu生效
exit
# 再次进入
wsl
```

## 三、配置SSL证书

### 3.1 申请免费SSL证书

1. 登录阿里云控制台
2. 搜索"SSL证书"
3. 申请免费证书（Let's Encrypt或DV证书）
4. 下载Nginx格式的证书文件

### 3.2 创建项目目录

```bash
# 在Ubuntu中创建项目目录
mkdir -p ~/delivery/nginx/ssl
mkdir -p ~/delivery/nginx/conf.d
cd ~/delivery
```

### 3.3 上传证书文件

**方法一：使用Windows资源管理器**

1. 在Windows资源管理器地址栏输入：`\\wsl$\Ubuntu\home\你的用户名\delivery\nginx\ssl`
2. 将证书文件复制到该目录：
   - `your-domain.com.pem`（或 `.crt`）
   - `your-domain.com.key`

**方法二：使用scp命令**

在Windows PowerShell中执行：
```powershell
scp 你的证书路径/your-domain.com.pem 你的用户名@localhost:/home/你的用户名/delivery/nginx/ssl/
scp 你的证书路径/your-domain.com.key 你的用户名@localhost:/home/你的用户名/delivery/nginx/ssl/
```

## 四、上传项目文件到云服务器

### 4.1 需要上传的文件清单

从本地开发电脑上传以下文件到云服务器：

```
mini_delivery/
├── docker-compose.yml          # Docker编排文件
├── server/                     # 后端项目
│   ├── Dockerfile.local        # 本地构建Dockerfile
│   ├── Dockerfile              # 生产环境Dockerfile
│   ├── pom.xml                 # Maven配置
│   └── src/                    # 源代码目录
├── nginx/                      # Nginx配置
│   ├── conf.d/
│   │   └── default.conf
│   └── ssl/
│       ├── your-domain.com.pem
│       └── your-domain.com.key
```

### 4.2 不需要上传的文件/目录

以下目录**不需要上传**，可以排除：

| 目录/文件        | 说明          | 原因                       |
| ---------------- | ------------- | -------------------------- |
| `server/target/` | Maven编译输出 | Docker构建时会重新编译生成 |
| `server/data/`   | 本地图片存储  | 生产环境使用Docker卷存储   |
| `miniprogram/`   | 小程序代码    | 通过微信开发者工具上传     |
| `开发计划.md`    | 开发文档      | 不影响运行                 |
| `开发需求.pdf`   | 需求文档      | 不影响运行                 |
| `*.md`           | 各类文档      | 不影响运行                 |

**重要说明**：
- `server/target/` 是Maven编译产物，Docker构建时会自动执行 `mvn package` 重新编译
- `server/data/` 是本地开发时存储上传图片的目录，生产环境图片存储在Docker卷 `upload_data` 中

## 五、配置Nginx

### 5.1 创建Nginx配置文件

```bash
nano ~/delivery/nginx/conf.d/default.conf
```

粘贴以下内容（替换 `your-domain.com` 为您的域名）：

```nginx
# HTTP 重定向到 HTTPS
server {
    listen 80;
    server_name your-domain.com;
    
    # 强制跳转HTTPS
    return 301 https://$server_name$request_uri;
}

# HTTPS 主配置
server {
    listen 443 ssl;
    server_name your-domain.com;

    # SSL证书配置
    ssl_certificate /etc/nginx/ssl/your-domain.com.pem;
    ssl_certificate_key /etc/nginx/ssl/your-domain.com.key;
    
    # SSL安全配置
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 10m;

    # 图片静态资源服务
    location /upload/ {
        alias /data/upload/;
        expires 30d;
        add_header Cache-Control "public, immutable";
        access_log off;
    }

    # 后端API代理
    location /api/ {
        proxy_pass http://backend:8081/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # 文件上传大小限制
        client_max_body_size 50M;
        
        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # 健康检查
    location /health {
        return 200 'OK';
        add_header Content-Type text/plain;
    }

    # Gzip压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
    gzip_min_length 1024;
}
```

按 `Ctrl+O` 保存，`Ctrl+X` 退出。

## 六、修改Docker Compose配置

### 6.1 修改docker-compose.yml

```bash
nano ~/delivery/docker-compose.yml
```

修改数据库密码等敏感信息：

```yaml
services:
  # MySQL 数据库
  mysql:
    image: mysql:8.0
    container_name: delivery-mysql
    restart: unless-stopped
    environment:
      MYSQL_ROOT_PASSWORD: 你的强密码    # 修改这里
      MYSQL_DATABASE: delivery
      TZ: Asia/Shanghai
    volumes:
      - mysql_data:/var/lib/mysql
      - ./server/src/main/resources/db/rebuild_database.sql:/docker-entrypoint-initdb.d/init.sql:ro
    command: --default-authentication-plugin=mysql_native_password --character-set-server=utf8mb4 --collation-server=utf8mb4_unicode_ci
    networks:
      - delivery-network
    healthcheck:
      test: [ "CMD", "mysqladmin", "ping", "-h", "localhost" ]
      interval: 10s
      timeout: 5s
      retries: 5

  # Spring Boot 后端服务
  backend:
    build:
      context: ./server
      dockerfile: Dockerfile.local
    container_name: delivery-backend
    restart: unless-stopped
    depends_on:
      mysql:
        condition: service_healthy
    environment:
      SPRING_DATASOURCE_URL: jdbc:mysql://mysql:3306/delivery?useUnicode=true&characterEncoding=utf-8&serverTimezone=Asia/Shanghai
      SPRING_DATASOURCE_USERNAME: root
      SPRING_DATASOURCE_PASSWORD: 你的强密码    # 与上面保持一致
      TZ: Asia/Shanghai
      FILE_UPLOAD_PATH: /data/upload/
    volumes:
      - upload_data:/data/upload
    networks:
      - delivery-network

  # Nginx 反向代理
  nginx:
    image: nginx:alpine
    container_name: delivery-nginx
    restart: unless-stopped
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx/conf.d:/etc/nginx/conf.d
      - ./nginx/ssl:/etc/nginx/ssl
      - upload_data:/data/upload:ro
    depends_on:
      - backend
    networks:
      - delivery-network

volumes:
  mysql_data:
  upload_data:

networks:
  delivery-network:
    driver: bridge
```

## 七、启动服务

提供两种部署方式，根据您的网络环境选择：

### 方式A：本地打包上传部署（推荐，无需梯子）

此方式在本地打包好JAR文件，上传到服务器直接运行，无需在服务器上下载Maven依赖。

#### A.1 在本地电脑打包

在**本地开发电脑**的PowerShell中执行：

```powershell
# 进入项目目录
cd C:\documents\development\mini_delivery\server

# 使用Maven打包（跳过测试）
mvn clean package -DskipTests

# 打包完成后，JAR文件位于：
# server/target/mini-delivery-1.0.0.jar
```

**如果没有安装Maven**，可以使用Maven Wrapper：

```powershell
# Windows使用
.\mvnw.cmd clean package -DskipTests
```

或者使用Docker在本地打包：

```powershell
# 使用Docker打包
docker run --rm -v ${PWD}:/app -w /app maven:3.9-eclipse-temurin-17 mvn clean package -DskipTests
```

#### A.2 上传JAR文件到服务器

```powershell
# 上传JAR文件
scp server/target/mini-delivery-1.0.0.jar ubuntu@服务器IP:/home/ubuntu/delivery/
```

#### A.3 在服务器上创建启动脚本

在服务器Ubuntu中执行：

```bash
# 创建启动脚本
nano ~/delivery/start.sh
```

粘贴以下内容：

```bash
#!/bin/bash

# 配置
APP_NAME="mini-delivery-1.0.0.jar"
APP_DIR="/home/ubuntu/delivery"
LOG_DIR="$APP_DIR/logs"
PID_FILE="$APP_DIR/app.pid"

# 数据库配置（根据实际情况修改）
DB_HOST="localhost"
DB_PORT="3306"
DB_NAME="delivery"
DB_USER="root"
DB_PASSWORD="你的数据库密码"

# 创建日志目录
mkdir -p $LOG_DIR

# 启动应用
cd $APP_DIR

nohup java -Xms512m -Xmx1024m \
  -Dspring.datasource.url="jdbc:mysql://${DB_HOST}:${DB_PORT}/${DB_NAME}?useUnicode=true&characterEncoding=utf-8&serverTimezone=Asia/Shanghai" \
  -Dspring.datasource.username=${DB_USER} \
  -Dspring.datasource.password=${DB_PASSWORD} \
  -Dfile.upload.path=/home/ubuntu/delivery/upload/ \
  -jar $APP_NAME \
  > $LOG_DIR/app.log 2>&1 &

echo $! > $PID_FILE
echo "应用已启动，PID: $(cat $PID_FILE)"
echo "日志文件: $LOG_DIR/app.log"
```

按 `Ctrl+O` 保存，`Ctrl+X` 退出。

```bash
# 添加执行权限
chmod +x ~/delivery/start.sh

# 创建停止脚本
nano ~/delivery/stop.sh
```

粘贴以下内容：

```bash
#!/bin/bash

APP_DIR="/home/ubuntu/delivery"
PID_FILE="$APP_DIR/app.pid"

if [ -f $PID_FILE ]; then
    PID=$(cat $PID_FILE)
    kill $PID
    rm $PID_FILE
    echo "应用已停止，PID: $PID"
else
    echo "未找到PID文件，应用可能未运行"
fi
```

按 `Ctrl+O` 保存，`Ctrl+X` 退出。

```bash
# 添加执行权限
chmod +x ~/delivery/stop.sh
```

#### A.7 配置Nginx反向代理

修改Nginx配置，将API请求转发到本地8081端口：

```bash
nano ~/delivery/nginx/conf.d/default.conf
```

修改 `location /api/` 部分：

```nginx
# 后端API代理
location /api/ {
    proxy_pass http://127.0.0.1:8081/api/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    
    client_max_body_size 50M;
    proxy_connect_timeout 60s;
    proxy_send_timeout 60s;
    proxy_read_timeout 60s;
}

# 图片静态资源服务
location /upload/ {
    alias /home/ubuntu/delivery/upload/;
    expires 30d;
    add_header Cache-Control "public, immutable";
    access_log off;
}
```

#### A.8 配置systemd服务（开机自启）

```bash
sudo nano /etc/systemd/system/delivery.service
```

粘贴以下内容：

```ini
[Unit]
Description=Delivery Management System
After=network.target mysql.service

[Service]
Type=forking
User=ubuntu
WorkingDirectory=/home/ubuntu/delivery
ExecStart=/home/ubuntu/delivery/start.sh
ExecStop=/home/ubuntu/delivery/stop.sh
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
```

启用服务：

```bash
# 重载systemd
sudo systemctl daemon-reload

# 启用开机自启
sudo systemctl enable delivery

# 启动服务
sudo systemctl start delivery

# 查看状态
sudo systemctl status delivery
```

## 十、图片存储配置

### 10.1 本地存储方案（已配置）

本项目默认使用本地文件存储，通过Nginx提供静态文件服务。

#### 存储架构

```
用户上传图片
    ↓
后端保存到 /data/upload/
    ↓
返回URL: /upload/xxx.png
    ↓
Nginx提供静态文件服务
    ↓
小程序访问: https://your-domain.com/upload/xxx.png
```

#### 数据持久化

图片存储在Docker卷 `upload_data` 中，即使容器重启也不会丢失。

#### 备份图片

```bash
# 备份图片到本地
sudo docker run --rm -v delivery_upload_data:/data/upload -v $(pwd)/backup:/backup alpine tar czf /backup/upload_backup_$(date +%Y%m%d).tar.gz -C /data/upload .

# 恢复备份
sudo docker run --rm -v delivery_upload_data:/data/upload -v $(pwd)/backup:/backup alpine tar xzf /backup/upload_backup_YYYYMMDD.tar.gz -C /data/upload
```

## 十二、常见问题

### 12.1 Docker服务未启动

```bash
# 检查Docker状态
sudo service docker status

# 启动Docker
sudo service docker start
```

### 12.2 端口被占用

```bash
# 查看端口占用
sudo lsof -i :80
sudo lsof -i :443

# 停止占用端口的进程
sudo kill -9 <PID>
```

### 12.3 域名未备案
- 解决：先完成域名备案，通常需要7-20天

### 12.4 SSL证书问题
- 检查证书是否正确安装
- 确保证书链完整
- 检查Nginx配置中的证书路径

### 12.5 请求域名不在合法列表
- 确认已在微信公众平台配置合法域名
- 域名必须以 `https://` 开头

### 12.6 WSL网络问题

如果外网无法访问：

```bash
# 检查WSL网络
ip addr show eth0

# 如果需要端口转发（在Windows PowerShell管理员中执行）
netsh interface portproxy add v4tov4 listenport=80 listenaddress=0.0.0.0 connectport=80 connectaddress=<WSL_IP>
netsh interface portproxy add v4tov4 listenport=443 listenaddress=0.0.0.0 connectport=443 connectaddress=<WSL_IP>

# 查看端口转发规则
netsh interface portproxy show all
```

## 十三、运维建议

### 13.1 定期备份数据库

```bash
# 创建备份脚本
cat > ~/delivery/backup.sh << 'EOF'
#!/bin/bash
BACKUP_DIR=~/delivery/backups
DATE=$(date +%Y%m%d_%H%M%S)
mkdir -p $BACKUP_DIR
sudo docker exec delivery-mysql mysqldump -u root -p你的密码 delivery > $BACKUP_DIR/delivery_$DATE.sql
# 删除7天前的备份
find $BACKUP_DIR -name "*.sql" -mtime +7 -delete
EOF

chmod +x ~/delivery/backup.sh

# 添加定时任务（每天凌晨2点备份）
(crontab -l 2>/dev/null; echo "0 2 * * * ~/delivery/backup.sh") | crontab -
```

```bash
# 查看容器状态

```
