# Mini Delivery - 微信小程序配送管理系统

## 一、项目概述

### 1.1 项目背景
这是一个微信小程序配送管理系统，用于货车司机向水果店和超市配送货品的管理。系统涉及四方角色，实现从店铺下单、订单分配、司机拣货到配送完成的完整流程管理。

### 1.2 角色定义

| 角色                   | 描述               | 核心功能                               |
| ---------------------- | ------------------ | -------------------------------------- |
| 总管理(ADMIN)          | 系统唯一超级管理员 | 分管理账号管理、全局数据监控           |
| 分管理(BRANCH_MANAGER) | 品牌方管理员       | 店铺/司机/商品管理、订单监控、账单生成 |
| 店铺(SHOP)             | 水果店/超市        | 商品浏览、下单、订单管理、账单支付     |
| 司机(DRIVER)           | 货车司机           | 订单分配、拣货、送货、导航             |

## 二、技术架构

### 2.1 技术选型

#### 前端（微信小程序）
- **框架**: 原生微信小程序
- **UI组件库**: 自定义组件
- **地图导航**: 腾讯地图 SDK

#### 后端
- **语言**: Java (Spring Boot 3.x)
- **数据库**: MySQL 8.0
- **ORM**: MyBatis-Plus
- **安全框架**: Spring Security + JWT
- **定时任务**: Spring Scheduled

#### 部署
- **容器化**: Docker + Docker Compose

```
待支付(0) → 待分配(1) → 待拣货(2) → 待送货(3) → 已完成(4)
                                    ↓
                               已取消(-1)
```

```
待支付(0) → 已支付(1)
```

## 六、本地部署方式

### 6.1 环境要求

- Docker Desktop
- 微信开发者工具

### 6.2 启动步骤

#### 1. 启动Docker服务

```bash
# 进入项目目录
cd mini_delivery

# 启动所有服务
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看后端日志
docker-compose logs -f backend
```

#### 2. 验证服务

```bash
# 测试后端API

```

#### 3. 配置小程序

打开 `miniprogram/config.js`，确认环境配置：

```javascript
const env = 'dev';  // 开发环境

const config = {
    dev: {
        baseUrl: 'http://localhost:8081/api',  // 本地开发
        tencentMapKey: ''
    },
    // ...
```

#### 4. 导入小程序项目

1. 打开微信开发者工具
2. 导入项目，选择 `miniprogram` 目录

## 七、真机调试指南

### 7.1 问题说明

校园网/公司网络通常启用了AP隔离，导致手机无法直接访问电脑的localhost。

### 7.2 解决方案

#### 方案1：使用手机热点（推荐）

1. 打开手机热点
2. 让电脑连接手机热点
3. 查看电脑新的IP地址：
   ```bash
   # Windows
   ipconfig
   
   # Mac/Linux
   ifconfig
   ```
4. 修改 `miniprogram/config.js` 中的 `baseUrl` 为新IP：
   ```javascript
   baseUrl: 'http://192.168.x.x:8081/api'
   ```
5. 确保手机和电脑连接同一热点

#### 方案2：使用真机调试2.0

1. 在微信开发者工具中，点击「真机调试」按钮旁边的下拉箭头
2. 选择「真机调试2.0」
3. 这个模式会自动处理网络穿透问题

#### 方案3：部署到云服务器

参考 [`生产环境部署指南.md`](生产环境部署指南.md) 进行部署。

### 7.3 防火墙配置

如果使用局域网IP仍无法访问，需要添加防火墙规则：

```bash
# Windows（以管理员身份运行）
netsh advfirewall firewall add rule name="Delivery Backend 8081" dir=in action=allow protocol=tcp localport=8081
```

## 十三、常见问题

### Q1: 数据库连接失败？
A: 检查 Docker 容器是否启动：`docker-compose ps`

### Q2: 小程序请求失败？
A: 
1. 检查 `miniprogram/config.js` 中的 `baseUrl` 配置
2. 确保微信开发者工具勾选了「不校验合法域名」

### Q3: 登录返回"用户不存在"？
A: 检查数据库中该用户的 `deleted` 字段是否为 1（已删除）

### Q4: 真机调试网络请求失败？
A: 参考「真机调试指南」章节，使用手机热点或真机调试2.0

### Q5: 文件上传失败？
A: 检查 `server/data/upload/` 目录是否存在且有写入权限

## 十四、开发规范

### 14.1 代码修改规范
- **不要**创建新的数据库初始化脚本
- **不要**修改 `rebuild_database.sql` 中的表结构（除非明确需求）
- **不要**随意添加新的实体类
- 修改代码前先阅读现有的实体类和服务类

### 14.2 前端开发规范
- 小程序页面位于 `miniprogram/pages/` 目录
- 全局配置在 `miniprogram/app.js`
- API请求使用 `app.request()` 方法

### 14.3 后端开发规范
- Controller 只负责请求处理，业务逻辑放在 Service
- 使用 MyBatis-Plus 的 LambdaQueryWrapper 进行查询
- 统一使用 Result 类返回响应
